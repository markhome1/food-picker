# 产品需求文档 - 今天吃啥

## 背景
和女朋友吃饭时选择困难，需要一个工具根据预算和距离快速选定餐厅。

## 目标
根据价格档位 + 距离范围筛选成都餐厅，支持随机推荐和列表浏览，记录每次就餐并生成统计报表。

## 非目标
- 云端部署（后续）
- 实时同步第三方平台数据
- 复杂推荐算法

## 目标用户
- 主要：你和女朋友（2人吃饭场景）

## 核心用户流程
1. **选餐厅**：选价格档位 → 选距离范围 → 随机推荐 / 列表浏览
2. **管理餐厅**：手动新增 / 编辑 / 删除 / CSV导入
3. **记录就餐**：选定餐厅 → 填出发地+消费+评价 → 保存
4. **查看统计**：总次数、每家去了几次、月度消费

## 功能需求
### P0 (Must Have)
- [x] 餐厅 CRUD + CSV 导入
- [x] 价格档位筛选（0-10/10-20/20-50/50-100/100+）
- [x] 距离范围筛选（全城/1km/3km/5km/10km）
- [x] 随机推荐一家
- [x] 列表浏览 + 排序
- [x] 就餐记录 CRUD
- [x] 统计看板
- [x] 高德 API 地理编码 + POI 搜索

### P1 (Should Have)
- [ ] 转盘动画效果
- [ ] 爬虫抓取（抖音/大众点评）
- [ ] 数据备份/导出

### P2 (Nice to Have)
- [ ] 小程序/App 发布
- [ ] 云端部署（PostgreSQL）
- [ ] 双人协作
